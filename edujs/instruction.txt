## 1. UX/UI & Visual Identity (The Instagram/TikTok Vibe)

To capture the addictive, clean, and highly visual feel of Instagram and TikTok, you should adopt a **Modern Dark/Light Hybrid UI** with high-contrast accents.

### Color Palette

* **Primary Background:** Deep Charcoal (`#121212`) for dark mode / Pure White (`#FFFFFF`) for light mode.
* **Cards/Containers:** Soft Off-Black (`#1E1E1E`) or Light Gray (`#F5F5F5`).
* **Accent 1 (Engagement - Like/Follow):** Neon Pink/Coral (`#FF007A`) — *TikTok inspired*.
* **Accent 2 (Knowledge/Tech/AI):** Electric Cyan/Blue (`#00F2FE`) — *Instagram/Tech inspired*.
* **Text:** Crisp White (`#FFFFFF`) or Dark Charcoal (`#222222`).

### UX Flow & Layout

* **The "Feed" Layout:** Vertical scrolling "cards" for mobile, and a multi-column grid for desktop. Each card represents a topic (e.g., a 1-minute summary of *Azure Fabric* or *TOGAF Architecture*).
* **Interactive Overlays:** Double-tap a card to "Like" (complete with a heart animation).
* **Micro-Interactions:** Smooth, springy transitions (using CSS `transition` or simple JS animation libraries) when switching between the Feed, Labs, and the Knowledge Graph.

## 2. Technical Architecture & Local Sharing (The Wi-Fi Question)

> **Can we use a browser in a home network so other family members on the same Wi-Fi can share and sync the JSON knowledge?**

**Yes, but with one major caveat:** Standard browser `localStorage` is strictly locked to the *specific device and browser* that created it. A family member on another device cannot read your phone's `localStorage` directly.

### The Solution: Peer-to-Peer (P2P) or Local Hosting

To keep this purely HTML/JS/CSS without buying a cloud database, you have three great options:

1. **The Export/Import Manual Sync (Easiest):** A "Sync" button downloads the updated Knowledge Graph as a `knowledge_graph.json` file. Another family member clicks "Import" on their device to merge it.
2. **Local Node.js Server (Recommended for true sync):** Run a tiny, 20-line Node.js script on one home computer. Everyone accesses the app via `http://[Your-Computer-IP]:3000`. The JSON is saved to a central file on that computer.
3. **WebRTC / Local Broadcast (Advanced Peer-to-Peer):** Use the browser’s WebRTC API to let browsers on the same network talk to each other directly and sync data.

## 3. Detailed Feature Specifications

### Module A: The Core Knowledge Graph

* **Data Structure:** A single JSON object containing `nodes` (topics like *PMP*, *Geometry*, *Azure AI*) and `edges` (connections, e.g., *Azure Databricks* "is used in" *Azure Fabric*).
* **Tech Hint:** Use a lightweight vanilla JS library like **Vis.js** or **Force-Directed Graphs via D3.js** to render the interactive, clickable graph.

### Module B: The Interactive Labs

These must be visual, using HTML5 Canvas or CSS animations.

| Lab Category | Interactive Feature UI |
| --- | --- |
| **Math Geometry** | A coordinate grid where users drag points to reshape triangles/circles; equations update in real-time. |
| **Physics Gravity** | A drop-zone where users adjust "Mass" and "Gravity" sliders to watch shapes fall or orbit, calculating velocity ($v = gt$). |
| **Biology Systems** | An SVG map of the human body. Clicking "Respiratory" highlights the lungs and plays a CSS animation of breathing. |
| **Code Viewer** | A split-screen panel. Left side: Raw HTML/JS code. Right side: Live preview box. Includes 5 pre-built templates showing how the UI interactions work. |

### Module C: The Chatbot AI Creator

Since you aren't using a heavy backend AI, you can build a **Rule-Based "Guided Wizard" Chatbot** that feels like an AI.

* **The Flow:** 1. Chatbot: *"Hey! What new topic did you learn today?"* 2. User inputs Title and Description.
3. Chatbot: *"Got it. Which existing topic does this connect to?"* (Shows a dropdown of current subjects).
4. User selects a topic.
5. **The Magic:** The JS script automatically creates a new node and an edge in your local JSON file, updating the Knowledge Graph instantly.
